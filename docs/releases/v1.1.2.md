# Release Notes — v1.1.2

## Summary
Switches `Badminton_MCP` from stdio to HTTP transport and deploys it to Render.com.

## Changes

### Transport: stdio → HTTP/SSE
- `Program.cs` now uses `WebApplication`, `.WithHttpTransport()`, and `app.MapMcp()`.
- Project SDK changed from `Microsoft.NET.Sdk` to `Microsoft.NET.Sdk.Web`.
- Package swapped from `ModelContextProtocol` to `ModelContextProtocol.AspNetCore 1.2.0`.

## New Infrastructure

### `Dockerfile.mcp`
Multi-stage Docker image for the MCP project. Listens on `$PORT` (Render) or `8080` (local).

### `render.yaml`
`badminton-mcp` web service added, pointing at `Dockerfile.mcp` with all required env-var placeholders.

### `.mcp.json.example`
Added `badminton-mcp-remote` entry with the Render URL template.

## Connecting to the Server
```json
{
  "type": "http",
  "url": "https://badminton-mcp.onrender.com/"
}
```
