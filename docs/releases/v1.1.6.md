# Release Notes — v1.1.6

## Summary
Implements the full notification service, APIs, player payment confirmation, and unpaid reminder trigger (BCM-108, BCM-109, BCM-110, BCM-111).

## New Features

### Notification Trigger Service (BCM-108)
`INotificationService` / `NotificationService` with three trigger methods:
- `TriggerPriceChangedAsync` — fires when session pricing is updated.
- `TriggerPaymentRecordedAsync` — fires when a player's payment is confirmed.
- `TriggerUnpaidReminderAsync` — fires for sessions with unpaid players; idempotent (one per session per day).

### Notification APIs (BCM-109)
`NotificationController` with four endpoints:

| Method | Route | Description |
|--------|-------|-------------|
| GET | `/api/notification` | Paginated list (`page`, `pageSize`) |
| GET | `/api/notification/unread-count` | Unread count for badge |
| PATCH | `/api/notification/{id}/read` | Mark one notification as read |
| PATCH | `/api/notification/read-all` | Mark all notifications as read |

### Player Confirm Payment (BCM-110)
- `POST /api/payment/session-player/{id}/confirm` — player signals they have paid; triggers a `PaymentRecorded` notification to the owner.

### Unpaid Reminder Manual Trigger (BCM-111)
- `POST /api/notification/trigger-reminder` — processes all `OnGoing` sessions and creates `UnpaidReminder` notifications. Skips sessions already reminded today. Returns `{ sessionsProcessed, notificationsCreated, skipped }`.
