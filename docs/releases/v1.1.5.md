# Release Notes — v1.1.5

## Summary
Adds the `Notification` data model and EF Core migration as the foundation for the notification system (BCM-107).

## New Features

### Notification Data Model
- New `NotificationType` enum: `PriceChanged`, `PaymentRecorded`, `UnpaidReminder` (stored as string in DB).
- New `Notification` entity implementing `IEntity` + `IUserOwnedEntity`:
  - `UserId` — tenant isolation.
  - `SessionId` — nullable FK to `Session` (`OnDelete: SetNull`).
  - `Type` — notification category.
  - `IsRead` — read/unread state.
  - `Payload` — JSON string for FE display (max 2000 chars).
- `DbSet<Notification>` registered in `AppDbContext` with:
  - Global `UserId` query filter.
  - Composite index `(UserId, IsRead)` for unread badge queries.
  - Composite index `(UserId, CreatedDate)` for timeline ordering.
- EF Core migration `AddNotification` generated.
