# Release Notes — v1.1.0

## Summary
Introduces the `Badminton_MCP` server, developer tooling, integration tests, and two backend bug fixes.

## New Features

### Badminton_MCP Server
A new .NET 10 MCP server project exposing Trello and Badminton API tools to GitHub Copilot via JSON-RPC over stdio.

- `TrelloClient` — Trello REST API v1 wrapper (member cards, board cards, board lists, move card, add comment).
- `TrelloTools` — five MCP tools:
  - `GetMyAICards` — returns open AI-labelled cards assigned to the configured member.
  - `GetBoardAICards` — returns all AI-labelled cards on the board, optionally filtered by member.
  - `GetBoardLists` — returns all lists on the board.
  - `MoveCardToList` — moves a card to a target list.
  - `AddCommentToCard` — posts a comment on a card.
- `BadmintonApiClient` — generic HTTP GET/POST wrapper for the Badminton REST API.

### Tests & Developer Tooling
- `Badminton_MCP.Tests` — xUnit integration test project targeting .NET 10.
- `TrelloConnectionTests` — 4 integration tests (skip gracefully without env vars).
- `trello.runsettings.example` — committed template for Trello credentials.

### Documentation
- `docs/version.md`, `docs/mcp-server.md`, `docs/mcp-workflow.md` added.

## Bug Fixes

### Member list missing stats (`GET /api/member`)
`GetMembersAsync()` returned `Wins`, `Losses`, `Draws`, and `WinRate` as `0` for every member. Fixed by calling `BuildMatchStatsAsync()` per member.

### Session registration not enforcing capacity
`AddMemberToSessionAsync()` had no slot check. Added `CountActiveBySessionAsync()` and a capacity guard blocking registration when the session is full.
