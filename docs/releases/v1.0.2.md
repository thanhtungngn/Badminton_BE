# v1.0.2 Release Notes

## New Features

### Update Individual Player Payment Amount
- **Endpoint:** `PUT /api/payment/session-player/{sessionPlayerId}/amount`
- **Body:** `{ "amountDue": <decimal> }`
- Allows the session owner to adjust the `AmountDue` for a specific player's payment record.
- `PaidStatus` is automatically re-evaluated after the update:
  - `Paid` if `AmountPaid >= AmountDue`
  - `Partial` if `AmountPaid > 0` but less than new `AmountDue`
  - `NotPaid` if `AmountPaid == 0`
- Returns `404` if no payment record exists for the given session player.
